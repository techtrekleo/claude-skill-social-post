**內容**：「寫 skill 寫到天亮，發現一件事：Claude 比咖啡有效☕」

## Case 5: Day 5 F11+F6b 反漏斗開源（2026-04-25 02:13）— 使用者 override 測試

**背景**：距 Day 1 僅 4 天，使用者 override R5（7 天內不 F6b）堅持 2:13 發。

**策略**：不純重複 F6b，混搭 F11 反漏斗 + F6b 結構。鉤子從「skill 自己發」改為「開源全部公式不用 star」。

**草稿重點**：
- 具體數字炸場（125,315 瀏覽 / +1,150 社群）
- 反漏斗 CTA「不用 star 不用追蹤」
- 連 Day 1 時段（2:13 AM）複製時段 buff
- 尾句輕量 meta「這篇也是 skill 排的」

**預測風險**：距 Day 1 太近 + F6b 續集 pattern → 可能複製 Day 2 的深化型（5-15K 觸及、80-150 讚、非追蹤者 < 30%）而非 Day 1 的擴散型 mega-viral。

### 19h vs 58h 數據演進（重要：早判會誤判）

| 指標 | 19h | **58h** |
|---|---|---|
| 瀏覽次數 | 186 | 314 |
| 瀏覽者 | 95 | **152** |
| 互動 | 23 | **40** |
| 讚 | 9 | 12 |
| 留言 | 2 | 2 |
| 分享（公開）| 2 | 2 |
| 非追蹤者 | 17.2% | **21%（66 人）** |
| **連結點擊** | **0** | **7** |
| 點擊次數 | 0 | 24 |
| GitHub stars 帶量 | ? | **+8**（22→30）|

⚠️ **教訓**：19h 太早下「fail」判斷。F11 反漏斗的 conversion 行為是長尾的，需要至少 48h 才看得到實際轉化。**Day 5 的觀察必須等 48-72h**。

### 對比 Day 1（同公式類、同時段、不同主題新鮮度）

| | Day 1 | Day 5 | 衰退 |
|---|---|---|---|
| 瀏覽者 | 74,510 | 95 | **-99.87%** |
| 讚 | 374 | 9 | -97.6% |
| 連結點擊 | 7 | 0 | -100% |

### 三大 learning（R5 cooldown 完全證實）

1. **主題重複比公式重複更致命**：F11 ≠ F6b（不同公式），但「promo skill」這**主題** 4 天內第二次 → 演算法判 self-promo 疲勞，不廣推
2. **02:13 時段 buff 不可重複用**：Day 1 buff 來自「新鮮主題 + 凌晨 + 演算法首推」三疊加；Day 5 只剩凌晨一項，無效
3. **F11 反漏斗 CTA 在鐵粉場失效**：連結點擊 0 = 鐵粉早知道 repo，反漏斗對他們無新意。F11 必須廣推到陌生人才有 authenticity 加分

### 但也不是純 fail（小確幸）

- 互動率 24% > 15% → 看到的人很投入
- 非追蹤者 17.2% > 15% → 比 Day 2-4 的 6-7% 都高，**輕微破圈**
- 2 個鐵粉手動分享、GitHub 可能帶 1-2 star

這篇實際是「給 Day 1 沒搶到的鐵粉的補課文」。

### 給 skill 的硬規則升級（R5 強化版）

**R5 強化版**：
- **同一「主題」4 天內不重複**（非公式重複，是主題重複）。即使換公式 / 換時段 / 換 hook 也救不了
- **主題定義**：標題或第一段提到的核心 noun phrase（例：skill / 開源 / Claude Code 社群 / 爆款公式）
- 違反成本：**最佳 = Day 5 級**（百觸及、單位數讚）；**最差** = 演算法降權影響後續貼文

下次使用者要「複製爆款」 → skill 必警告：「Day 5 同樣嘗試只剩 Day 1 的 0.13% 觸及。建議：(1) 換主題、或 (2) 等 7 天後再來。」

### 演算法 second push（重大發現）

1. **R5 主題冷卻定義精緻化**：不是「公式重複」會被罰，是「**敘事意圖重複**」會被罰

   - 修正：衰減主因是「敘事意圖重複」而非「hype 詞語本身」

3. **F6b 兩變體公式定型**：
   - 變體 A：promo 型（self-evident 鉤子）— Day 1 mega-viral 範本
   - 變體 B：復盤型（self-experimental 鉤子）— Day 6 突破鐵粉圈範本

### 獨立 insight 詳述（無 SKILL.md 對應的 meta 規則）

#### 歸納 2（擴散型 vs 深化型）

兩種貼文都成功但功能不同，**不可用同標準比較**：
- 擴散型（Day 1 模式）：高觸及、高分享、低 conversion
- 深化型（Day 2 模式）：低觸及、高互動率、高 conversion

#### 歸納 6（使用者 override pattern，meta 規則）

使用者堅持 override skill 規則時：
- **尊重使用者決定**，執行 + 標記 override 在戰績備註
- **不重複爭論 risk**（一次說明即可）
- **降風險策略**：若 override 的是公式類規則，嘗試混搭其他公式（例 Day 5 將 F6b 跟 F11 混搭）
- **戰績出來後實證**：無論成功失敗，都是 skill 迭代材料（寫回本檔當 case study）
- 私人版 SKILL.md「安全閘」段落允許 session-scoped bypass「確認」字眼；開源版永不 bypass

#### 歸納 10（Viral 4 條件公式 detailed）

```
viral = 結構 + voice + 全新敘事意圖（4 天 + 月配額）+ 時段
```

| 案例 | 結構 | voice | 新意圖 | 時段 | 結果 |
